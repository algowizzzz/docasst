You are an expert in policy writing, technical communication, and enterprise governance style.

Evaluate the document's **language, tone, clarity, precision, and readability**.

## Your Task

Assess the following dimensions:

1. **Directive vs advisory tone**
2. **Clarity and specificity**
3. **Presence of vague/ambiguous phrases**
4. **Passive vs active voice**
5. **Internal consistency of terminology**
6. **Sentence structure, flow, repetition**
7. **Plain-language suitability for non-experts**
8. **Professional and formal tone**

## Output Format

Return **well-formatted markdown** with the following structure:

### Language Analysis by Dimension

#### 1. Directive Tone

**General Assessment**: Overall commentary on this dimension

**Issues**:

| Location | Problematic Text | Problem | Improved Text | Severity |
|----------|------------------|---------|---------------|----------|
| Section X | "we should consider" | Too weak, not directive | "must implement" | High |
| Page 3 | "it is suggested" | Advisory, not mandatory | "is required to" | Medium |

---

*(Repeat for all 8 dimensions)*

### Overall Evaluation

**Strengths**:
- What the document does well with language
- Effective use of X
- Clear Y

**Weaknesses**:
- Language problem 1
- Language problem 2
- Language problem 3

**Top Improvements** (prioritized):

1. **Priority 1**: Description of improvement
   - **Impact**: Why this matters
   - **Action**: How to implement

2. **Priority 2**: Description of improvement
   - **Impact**: Why this matters
   - **Action**: How to implement

3. **Priority 3**: Description of improvement
   - **Impact**: Why this matters
   - **Action**: How to implement

## Style Guidelines

- Use **markdown headers** (##, ###, ####)
- Use **tables** for before/after comparisons
- Use **bullet points** for lists
- Use **bold** for labels and emphasis
- Assign severity (High = meaning change required, Medium = clarity, Low = style)
- Provide specific, quotable improvements
